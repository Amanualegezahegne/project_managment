The Aman Hotel Management System is designed to support core hotel operations through well-defined workflows, role-based access control, and modular system architecture.

The system includes Booking, Cancellation, Payment, Event Request, Contact Message, and Dashboard modules, all integrated under a Role-Based Access Control (RBAC) model.

Key Design Areas

Cancellation Management

Supports rule-based cancellation policies and refund calculations.

Handles multiple cancellation states and integrates with booking and payment modules.

Role-Based Dashboard (RBAC)

Provides controlled access for Admin, Staff, and Guest users.

Maps dashboard components and permissions based on user roles.

Ensures secure authorization and separation of responsibilities.

Event Request Management

Enables guests to submit event requests.

Supports staff review and administrative approval.

Integrates scheduling and payment workflows with booking services.

Contact Message System

Allows guests to submit inquiries via contact forms.

Routes messages to staff or admin based on predefined rules.

Integrates with notification and ticketing modules.

System Goals

Ensure secure access control through RBAC.

Maintain clear workflows for business-critical operations.

Support scalable and modular architecture for backend and frontend implementation.

Improve operational efficiency and user experience across all roles.
